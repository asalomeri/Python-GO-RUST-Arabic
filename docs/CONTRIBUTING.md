# دليل المساهمة – CONTRIBUTING

شكرًا لرغبتك في المساهمة في مستودع **Python-GO-RUST-Arabic**!  
هذا الدليل يشرح الطريقة الموصى بها للعمل بشكل منظم ومتناسق مع بقية المساهمين.

---

## 1. قبل البدء بأي تغيير كبير

افتح **Issue** أولاً لمناقشة الفكرة قبل كتابة الكود:

1. تحقق من [القضايا المفتوحة](../../issues) لتجنب التكرار.
2. افتح Issue جديداً باستخدام أحد القوالب المتاحة (Bug Report أو Feature Request).
3. انتظر تعليقاً من المشرفين قبل البدء بتغيير جوهري.

---

## 2. إعداد بيئة التطوير

تأكد من توفر الأدوات التالية:

| اللغة  | الإصدار الأدنى المقترح |
|--------|------------------------|
| Python | 3.10+                  |
| Go     | 1.21+                  |
| Rust   | stable (عبر rustup)    |
| Git    | أي إصدار حديث          |

---

## 3. تسمية الفروع (Branch Naming)

استخدم الصيغة: `<نوع>/<وصف-قصير-بالإنجليزية>`

| النوع    | متى يُستخدم                        | مثال                         |
|----------|------------------------------------|------------------------------|
| `feat`   | ميزة جديدة أو محتوى تعليمي جديد   | `feat/add-rust-closures`     |
| `fix`    | إصلاح خطأ في كود أو توثيق          | `fix/python-calculator-bug`  |
| `docs`   | تحسين التوثيق فقط                  | `docs/update-setup-guide`    |
| `chore`  | صيانة، تنظيم، تحديث إعدادات        | `chore/update-dependabot`    |

---

## 4. أسلوب رسائل الـ Commit

اتبع صيغة [Conventional Commits](https://www.conventionalcommits.org/) المبسّطة:

```
<نوع>: <وصف قصير بالعربية أو الإنجليزية>
```

**أمثلة:**
```
feat: إضافة مثال على Goroutines في Go
fix: تصحيح خطأ في دالة الجمع بالحاسبة
docs: تحديث دليل الإعداد SETUP.md
chore: تحديث إعدادات Dependabot
```

---

## 5. تشغيل الاختبارات محلياً

### Python
```bash
# تثبيت المتطلبات
pip install -r requirements.txt

# تشغيل الاختبارات
pytest tests/python_tests.py -v

# تنسيق الكود
black python/

# فحص الجودة
pylint python/
```

### Go
```bash
# من جذر المستودع أو داخل مجلد go/
go test ./...

# تنسيق الكود
gofmt -w .
```

### Rust
```bash
# من جذر المستودع (حيث Cargo.toml)
cargo test

# تنسيق الكود
cargo fmt

# فحص الجودة
cargo clippy
```

---

## 6. فتح Pull Request وربطه بالـ Issue

1. **ادفع فرعك** إلى GitHub:
   ```bash
   git push -u origin feat/اسم-ميزتك
   ```

2. **افتح PR** نحو فرع `main` من واجهة GitHub.

3. **اربط PR بالـ Issue** بإضافة هذا في وصف الـ PR:
   ```
   Closes #رقم-الـ-Issue
   ```

4. **تأكد** من اجتياز جميع فحوصات الـ CI قبل طلب المراجعة.

5. **استجب** لأي تعليقات من المراجعين وعدّل الكود إذا طُلب.

---

## 7. قواعد السلوك

- احترم جميع المساهمين في النقاشات والتعليقات.
- قدّم ملاحظات بنّاءة وموضوعية.
- إذا كان لديك سؤال، لا تتردد في طرحه عبر Issues.
